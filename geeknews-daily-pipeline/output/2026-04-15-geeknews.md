## 3. 단 20MB HTTP 패킷으로 Django 서버를 1분간 먹통 만드는 취약점이 공개되었습니다 (CVE-2026-33033)

🔗 [원문 보기](https://new-blog.ch4n3.kr/django-pre-auth-denial-of-service-kr/)

### 핵심 요약

- Django의 MultiPartParser에서 발생하는 취약점(CVE-2026-33033) 공개.
- 특정 조건에서 20MB의 HTTP 패킷으로 서버를 1분간 다운시킬 수 있음.
- 취약점은 Content-Transfer-Encoding: base64 파트 본문이 공백으로 구성될 때 발생.
- 이로 인해 서버의 CPU 자원이 고갈되어 서비스 거부 상태로 전환됨.
- 개발자들은 해당 취약점을 패치하거나 방어책을 마련해야 함.

### 💡 왜 중요한가

이 기사는 Django 서버를 사용하는 개발자들에게 심각한 서비스 거부 공격의 가능성을 경고하며, 즉각적인 대응이 필요함을 강조합니다. 보안 취약점에 대한 인식을 높이고, 예방 조치를 마련하는 것이 중요합니다.
