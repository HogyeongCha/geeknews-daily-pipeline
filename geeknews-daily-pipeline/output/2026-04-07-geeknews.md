🔗 [원문 보기](https://blog.pypi.org/posts/2026-04-02-incident-report-litellm-telnyx-supply-chain-attack/)

### 핵심 요약

- PyPI 보안팀은 LiteLLM과 Telnyx 패키지에서 발생한 공급망 공격 사건을 보고했다.
- 공격자는 Trivy의 의존성 취약점을 이용해 API 토큰을 탈취하였다.
- 탈취된 API 토큰을 통해 악성 패키지 버전이 PyPI에 배포되었다.
- PyPI는 이러한 공격을 방지하기 위해 여러 보안 조치를 강화할 계획이다.
- 사용자들은 신뢰할 수 있는 패키지 소스를 사용하는 것이 중요하다.

### 💡 왜 중요한가

이 기사는 개발자와 IT 종사자들에게 공급망 공격의 위험성을 경고하며, 패키지 관리와 보안에 대한 경각심을 높이는 데 중요한 정보를 제공한다. 또한, 보안 강화 조치에 대한 이해는 안전한 개발 환경을 유지하는 데 필수적이다.
